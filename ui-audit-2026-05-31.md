# UI Audit Notes - 2026-05-31

## Context

Audit and first cleanup pass for the Budgeting Dashboard UI. The goal was to identify what felt AI-generated, make light and medium-risk improvements, and preserve the existing product structure.

Primary register: product UI. The dashboard should feel like a clear personal finance workspace, not a decorative fintech landing page.

## Original Audit Summary

Overall score: 14/20, good but generic.

| Dimension | Score | Main finding |
|---|---:|---|
| Accessibility | 3/4 | Good primitives and skip link, but some icon-only or hover-only actions needed stronger accessibility. |
| Performance | 3/4 | Build passed, but Vite still warns about large app and Recharts chunks. |
| Theming | 2/4 | Tokens existed, but chart colors, aliases, gradients, and hard-coded colors were inconsistent. |
| Responsive | 3/4 | Mobile nav/cards are solid; filters and marketing surfaces still need polish. |
| Anti-patterns | 3/4 dashboard, 1/4 marketing | Dashboard is acceptable; landing/auth had stronger template signals. |

## What Felt AI-Generated

- Inter + Plus Jakarta Sans + Playfair font mix felt like a generated premium SaaS pattern.
- Gradient text, glass utilities, glow aliases, and `blurple`/`teal` legacy naming were strong scaffold signals.
- Landing/auth pages used generic copy such as "Your financial journey starts here" and decorative emphasis.
- KPI hero cards combined big radius, border, gradient, large shadow, and decorative watermark icons.
- Some transaction cards used a side accent stripe pattern.
- Chart colors had unclear meaning: savings was grey in tokens, while pages hard-coded teal, blue, purple, amber, and grey.

  - Accounts
  - Categories

  - Transactions
  - Dividend calculator rows

### OAuth Avatar Fix

## Still Missing / Heavy Changes

These are intentionally not done yet because they are closer to redesign work.

1. Landing page redesign

4. Dashboard density and hierarchy pass

   - Make KPI groups more scannable and less card-heavy.

5. Chart system completion
   - Apply semantic chart colors to all remaining investment/calculator charts.
   - Add chart legends or labels that explain the meaning of repeated colors.

6. Performance/bundling pass
   - Split large shared chunks.
   - Consider manual chunks for Recharts and Radix-heavy UI.
   - Review lazy loading boundaries for chart-heavy dashboard routes.
